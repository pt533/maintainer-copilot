# Security Policy

## Supported versions

This project is currently early-stage and under active development.
Security fixes will be applied to the latest version on the default branch.

## Reporting a vulnerability

Please do not report security vulnerabilities in public issues.

Instead:
- open a private vulnerability report if GitHub private reporting is enabled
- or contact the maintainer through GitHub and clearly label the report as a security issue

Please include:
- a description of the issue
- reproduction steps if available
- impact assessment
- suggested mitigation if known

## What to avoid

Do not include secrets, API keys, tokens, or private credentials in issues or pull requests.
